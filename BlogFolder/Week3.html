Working through the slides for Interaction Design for the web, define and discuss your user alignment, and map out your site's content, experience, and interface elements.

User Alignment
What I want the user to see and do:
I want the user to “play around” on the website. In the sense that I want to user to be engaged and have fun with particularly the various interface elements that allow for engaging user interaction. I want to user to spend time experimenting with the interface elements. I want the user to feel at ease while using the website even as they have fun – I want it to be a relaxing and refreshing experience. I want the user to explore my portfolio and engage with and give feedback to my Essays, Blogs and projects.

My target audience and their requirements:
Software developer employers who are looking for someone who is willing to take on challenges and experiment with unconventional and creative interface elements. The average user who just wants to have a “fun” time playing around with my website.

My content (what is interesting and important):
My portfolio (my work and skills), Blogs (my process through this fluctuating journey of creating my first official html website), Essays (my objective analysis and personal opinions on the topics given) and profile (to get to know me) is majority of the content. The most important being my portfolio as it will reflect my skills, my capabilities, and interests.

    Reflect on how the exercise above relates to what you know about your structure and URL scheme and how it relates to semantic markup.

    The exercise above helped me revisit my folder structure and URL scheme which also in turn relates to the semantic markup. By clearly defining the user alignment, I was able to see how, through interface elements, I can direct my users to view the information I want them to see but also helped me to realize that through my folder structure and URL scheme, content should be easily accessible in order for users to have a pleasant experience. If my folder structure and URL scheme is simple and straightforward, so will the user experience be satisfying.

    Semantic markup is a part of creating this user experience because semantic markup is about how the HTML elements are used appropriately for their intending purpose to express the interface elements accordingly. By embedding tags correctly and using the appropriate tags, the browser will convey to user my designs as I intend and therefore the user can respond and interact appropriately without getting confused. For example, I will try avoiding 'div' and 'span' tags as a means of separating content on the page because they do not convey much meaning, and I will instead opt for 'article' and 'section' tags to section parts of my page.
