Wireframes

HOMEPAGE:

The homepage is simple and only contains 4 main components:

  • The main navigation area which contains the links to the main pages of the website. It is situated at the top in large writing for each of access.
  • A text area for introductory text that has a "typing animation" to welcome the user to the website.
  • A group of draggable letter elements that are randomly spawned in the middle area. Users upon visiting the website interact immediately giving them a sense of control and to pique their curiosity.
  • A footer area that contains my primary socials.

    • Navigation Homepages:

    All navigation pages have the same layout. The only difference is the colour scheme (further explained in the style guide). Tha website title is situated in the top left corner, the main navigation in the top right corner. Together these two elements form the header.

    Below the header, large text with "typing animation" display the heading of the homepage that the user is on.

    Below the heading, a short summary of that specific group of pages is given, to give the user context into what the subpages will be about.

    Below the summary, a secondary navigation to specfic subpages is displayed vertically in large boxes. Each "box" has the date the page was posted, a heading, a short summary of the page, and a "Read More" button. User's can decide which posts they want to read based off the date, title and summaries.

    At the bottom of the page, there is a footer - the same as the one on the homepage

    Navigation Subpages:

    Each subpage - similar to the homepages - has the same layout with different colours. At the top the header is situated.

    On the right, the secondary navigation to the other subpages is hidden initially to not disturb the user, but it is visisble when the user presses the button that says "Other..." The button is always visible to the user no matter how far they have scrolled down for ease of access and convience should the user wish to navigate to a different post quickly.

    On the left, there is a button to take the user back to the specfic homepage should the user wish to.

    Below this button is the title of the post in big bold letters. Below the title is the author and date of the post, and below that is the content of the post itself.

    At the bottom of the page, there is a scroll up button should the user wish to go back to the top of the page, and below this button is the footer.

    Portfolio Page:

    The portfolio page uses the old idea of the infinte horizontal scroll that was initially for the specific homepages.

    There is a heading, and a short description of the portfolio projects

    Underneath this description is a row of buttons that give the user the option to filter the projects by multiple groups. The user can easily find the projects there are interested in.

    Each project in the horizontal scroll has an image, title with hyperlink to the respective project, and a text summary of the project. And underneath, the footer is displayed at the bottom of the page.

    Profile Page:

    The profile page is similar to the old wireframe. Each grouping of information is displayed as a draggable box that the user can interact with, similar to the letters on the homepage. The profile page is more casual than other parts of website hence the interactable objects. The addition however, is in the implementation of the flip cards that was originally for the blog posts. The flip card function is more appropriate in this section. And so each box is draggable and flippable.

    Another interactable addition is the implementation of a random fact generator on one of the cards.The user can have fun reading interesting facts about me.

    Information Structure

    The final information structure and hierarchy of the website is as shown in the image below

    Homepage:

    The user is met first with the homepage.From here the user can navigate to the blogs, essays, profile, or portfolio. In the footer, the user can also navigate to various socials, and can directly send an email should they wish to.

    Navigation pages:

    The user can navigate from the homepage, to various navigation pages namely, the blog nav page, the essay nav page, and the designs nav page. All these nav pages have the same layout with the same interface elements to keep the website consistent and uniform.

    Subpages:

    The user can navigate from the nav pages to the various subpages. The interface elements across the subpages are the same. The subpages also allow the user to navigate to other main pages, the homepage of that subpage, and other subpages (within the same navigation category).

    Profile:

    The profile is the only page that has slightly different interface elements. I reintroduced the draggable elements and made it central to how the user interacts with the page.

    Portfolio:

    The portfolio page has the same interface elements as the nav pages, however, I might consider changing the horizontal scroll bar to be a vertical infinite scroll bar to showcase my portfolio works instead – but at the moment it is the same horizontal scroll bar as in the navigation pages.

    Overall however, the way the infomation is structured has not majorly changed since my first wireframes.

    User Flow

    The profile page is slightly different to other parts the website. I chose to reintroduce the moveable images and in addition moveable text areas/boxes. I decided to reintroduce this means of user interactivity because the profile page is more personal and less formal and thus to give the user a sense of my personality, the moveable sections make for a better user experience.

    Style Guide

    This is the general layout for both subpages that showcase my designs. Similar to the Blogs subpage, the image is on the left and the text on the right to keep consistent. Each subpage has a button navigating to the latter subpage.
