LAST ITERATION

this was posted by Ivy Chepkwony on 23rd June 2024

Goals

The goal for this website is promote and encourage user control and interactivity. The name of the website is "Playground" and thus to stay true to this, I aim to give the user a truly "playful" and "enjoyable" experience by introducing them to interactive elements, and giving meaningful feedback.

Wireframes

HOMEPAGE:

The homepage is simple and only contains 4 main components:

  • The main navigation area which contains the links to the main pages of the website. It is situated at the top in large writing for each of access.
  • A text area for introductory text that has a "typing animation" to welcome the user to the website.
  • A group of draggable letter elements that are randomly spawned in the middle area. Users upon visiting the website interact immediately giving them a sense of control and to pique their curiosity.
  • A footer area that contains my primary socials.

    • Navigation Homepages:

    All navigation pages have the same layout. The only difference is the colour scheme (further explained in the style guide). Tha website title is situated in the top left corner, the main navigation in the top right corner. Together these two elements form the header.

    Below the header, large text with "typing animation" display the heading of the homepage that the user is on.

    Below the heading, a short summary of that specific group of pages is given, to give the user context into what the subpages will be about.

    Below the summary, a secondary navigation to specfic subpages is displayed vertically in large boxes. Each "box" has the date the page was posted, a heading, a short summary of the page, and a "Read More" button. User's can decide which posts they want to read based off the date, title and summaries.

    At the bottom of the page, there is a footer - the same as the one on the homepage

    Navigation Subpages:

    Each subpage - similar to the homepages - has the same layout with different colours. At the top the header is situated.

    On the right, the secondary navigation to the other subpages is hidden initially to not disturb the user, but it is visisble when the user presses the button that says "Other..." The button is always visible to the user no matter how far they have scrolled down for ease of access and convience should the user wish to navigate to a different post quickly.

    On the left, there is a button to take the user back to the specfic homepage should the user wish to.

    Below this button is the title of the post in big bold letters. Below the title is the author and date of the post, and below that is the content of the post itself.

    At the bottom of the page, there is a scroll up button should the user wish to go back to the top of the page, and below this button is the footer.

    Portfolio Page:

    The portfolio page uses the old idea of the infinte horizontal scroll that was initially for the specific homepages.

    There is a heading, and a short description of the portfolio projects

    Underneath this description is a row of buttons that give the user the option to filter the projects by multiple groups. The user can easily find the projects there are interested in.

    Each project in the horizontal scroll has an image, title with hyperlink to the respective project, and a text summary of the project. And underneath, the footer is displayed at the bottom of the page.

    Profile Page:

    The profile page is similar to the old wireframe. Each grouping of information is displayed as a draggable box that the user can interact with, similar to the letters on the homepage. The profile page is more casual than other parts of website hence the interactable objects. The addition however, is in the implementation of the flip cards that was originally for the blog posts. The flip card function is more appropriate in this section. And so each box is draggable and flippable.

    Another interactable addition is the implementation of a random fact generator on one of the cards.The user can have fun reading interesting facts about me.

    Information Structure

    The final information structure and hierarchy of the website is as shown in the image below

    Homepage:

    The user is met first with the homepage.From here the user can navigate to the blogs, essays, profile, or portfolio. In the footer, the user can also navigate to various socials, and can directly send an email should they wish to.

    Navigation pages:

    The user can navigate from the homepage, to various navigation pages namely, the blog nav page, the essay nav page, and the designs nav page. All these nav pages have the same layout with the same interface elements to keep the website consistent and uniform.

    Subpages:

    The user can navigate from the nav pages to the various subpages. The interface elements across the subpages are the same. The subpages also allow the user to navigate to other main pages, the homepage of that subpage, and other subpages (within the same navigation category).

    Profile:

    The profile is the only page that has slightly different interface elements. I reintroduced the draggable elements and made it central to how the user interacts with the page. I also used the "flip card" function here as described earlier.

    Portfolio:

    The user can navigate to the specfic locations where the respective projects are uploaded by hovering over the title of the project. The user can also navigate to other pages by the main navigation.

    User Flow

    The User Flow is fairly similar to the first two iterations such that the 3-click rule is generally adhered to. However, there are no Portfolio Subpages, instead the user is taken directly to the repos or website in which the projects are uploaded on. In addition, the user can navigate directly to the social links provided in the footer which was not in the original user flow.

    Interface Elements

    Input Controls

    Allow user to input information

    Buttons

    In the contact form, the user presses the submit button when sending a message request to me.

    Drop Down Navigation

    In the secondary navigation, in the subpages, the user presses the button to reveal information about te secondary navigation.

    The custom cursor is not only for visual aesthetic, but the size of the custom cursor was increased, to make navigating the website easier and more accessible. This goes hand-in-hand with the in-built pointer and grab cursor when the user drags elements of clicks specific elements.

    Style Guide

    Colours

    The colour palette was inspired by common colours found on actual playgrounds [1]. To truly make it a playful experence, I experimented with bold colours (colours of the rainbow), and used them strategically in such a way to bring life and excitement to the website, without overwhelming the user. Therefore the bold colours were balanced with the azure blue and the black. The Azure blue was used instead of plain white simply because white was too contrasting, the azure blue was a more cool toned colour that contrasts with the black, yet it is not overwhleming on the eyes. Even the shade of black used was toned down slightly to be easier on the eyes. Each colour (red, orange, blue, green, pink, purple) was used in a different section of the website. The colours were used only for highlights,headings, nav componenets, buttons, and links in the footer. In this way the colours were used in parts of the website that were most important, or parts that i needed to stand out. I tried to also spread the positioning of the colours across the page to have some balance. Most of the text is black, for ease of reading, except for areas where an element was highlighted and thus the colour changed with the hover effect.

  • The user is greeted with green, because green on playgrounds is a grounding colour and brings tranquility. As the user enters the website they are greeted with an inviting colour.

  • Pink is used for the profile page because it is described as a youthful and sweet colour in addition to the fact that it is my favourite colour. This colour showcases my personality the most.

  • Purple is a more mature, elegant colour, hence why it was used to showcase my portfolio projects.

  • The orange, blue and red colours on the other pages do not have as in-depth meaning, however, orange and red are particularly passionate colours hence appropriate for my blog posts and essays since they reflect my thoughts. The designs are in blue because the designs are reflective of a more analytical thought process and thus the colour blue generally invokes a more neutral, calm emotions than the other colours.

    • Fonts and Text

    The fonts used are derived from the "Lucida Console", "Courier New" font family. The Lucida Console is an evenly spaced font type, and similarly, the "Courier New" is commonly used in code and programming environments. These fonts are therefore legible, clean looking fonts that make reading easy and comfortable. Part of creating an enjoyable user experience is ensuring that users can comfortably digest the information fed to them.

    The space between the lines are also slightly larger than the standard line-spacing. This makes reading easier for users that have cognitive limitations.

    The sizing of the headings on the various pages was intentional. The headings are purposefully in large text around the website for legibility purposes.

    Styling of Elements

    Most elements on the website have rounded edges because as breifly mentioned in previous iterations, the website is meant to be more playful than serious, and thus, the rounded borders are more friendly and less serious.

    Some elements when hovered upon enlarge and/or change colours, this is for user responsiveness to make the user experience more enjoyable.
